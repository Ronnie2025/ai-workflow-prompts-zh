# AI 工作流提示词库（中文免费版）

使用方法：复制对应提示词，粘贴你的真实材料。第一次运行后，把效果好的输入格式、输出格式和检查点保存下来，下次只替换材料。

## 1. 工作流诊断

```text
你是一个工作流诊断顾问。

请帮我分析下面这个工作流程：

【当前流程】
粘贴流程描述

请输出：
1. 这个流程的输入是什么
2. 这个流程的输出是什么
3. 哪些步骤是重复劳动
4. 哪些步骤必须保留人工判断
5. 哪些步骤可以交给 AI 辅助
6. 最应该优先改造的 3 个环节
7. 第一版工作流应该如何命名
```

适合：开始搭 AI 工作台前，先判断哪个流程值得固定。

## 2. 素材整理

```text
你是一个资料整理助手。

我会给你一批原始素材，请你不要急着写成品，先按以下结构整理：

1. 事实信息
2. 观点判断
3. 可引用案例
4. 可展开选题
5. 需要补充确认的问题

输出用 Markdown 表格。
遇到不确定信息请标注“待确认”，不要编造。
```

适合：把聊天记录、笔记、访谈和资料包先变成可处理材料。

## 3. 长文转小红书图文

```text
你是一个小红书图文策划。

请把下面这篇长文拆成 1 组 3:4 图文脚本。

要求：
- 8 页以内
- 每页一个核心信息
- 标题直接，不使用夸张承诺
- 适合知识型读者
- 最后一页给行动建议

输出格式：
页码｜页面标题｜页面正文｜配图建议
```

适合：课程笔记、长文、复盘、访谈整理成图文草稿。

## 4. 小红书标题生成

```text
你是一个小红书标题编辑。

请基于下面的正文内容生成 12 个标题。

要求：
- 不使用夸张承诺
- 不制造焦虑
- 不写虚假数字
- 每个标题都要让读者知道能获得什么
- 分成 4 类：问题型、结果型、场景型、反常识型

输出格式：
类型｜标题｜适合读者｜风险提醒
```

适合：从同一篇内容里测试不同标题角度。

## 5. 会议记录转行动清单

```text
你是一个项目经理。

请把下面会议记录整理成行动清单。

只输出：
1. 已决定事项
2. 待确认问题
3. 责任人
4. 截止时间
5. 下次会议前必须完成的动作

如果原文没有责任人或时间，请标记为“待确认”，不要编造。
```

适合：会后马上拆行动项，减少遗漏。

## 6. 客户资料转方案初稿

```text
你是一个 B2B 解决方案顾问。

请根据客户资料输出一版方案初稿。

要求：
- 先还原客户真实需求
- 再判断当前问题属于业务问题、流程问题、数据问题还是工具问题
- 不要直接堆功能
- 输出一页可汇报结构

输出：
1. 客户背景
2. 核心问题
3. 现有阻碍
4. 建议方案
5. 交付边界
6. 下一步动作
```

适合：销售、咨询、客户成功、项目负责人整理方案初稿。

## 7. PPT 大纲生成

```text
你是一个商业汇报 PPT 策划。

请把下面材料整理成 PPT 大纲。

要求：
- 先给整体叙事线
- 再按页输出
- 每页只表达一个结论
- 不要写成论文

输出格式：
页码｜页面标题｜核心结论｜内容要点｜建议图表
```

适合：汇报、课程、提案、复盘先搭页级结构。

## 8. 课程大纲生成

```text
你是一个课程产品设计师。

请根据下面主题设计一版课程大纲。

要求：
- 面向明确人群
- 每节课解决一个具体问题
- 不堆概念
- 每节课都要有练习或作业

输出：
1. 目标学员
2. 学完能做到什么
3. 课程模块
4. 每节课标题
5. 每节课交付物
6. 课后作业
```

适合：把经验型知识整理成课程结构。

```text
要求：

- 给出优秀作业标准
- 给出常见错误

输出：
1. 作业目标
2. 操作步骤
3. 提交格式
4. 评分标准
5. 常见错误
```

适合：课程老师把讲义变成可执行练习。

```text

要求：
- 相似问题合并
- 问题用用户原话风格
- 回答要直接
- 标出需要人工补充的地方

输出格式：
问题｜标准回答｜适用场景｜待补信息
```

适合：把碎片化社群答疑沉淀成知识库。

## 11. 周报生成

```text
你是一个项目运营经理。

请根据下面的本周记录生成一份周报。

要求：
- 不写空话
- 先讲结果，再讲动作
- 区分已完成、进行中、风险、下周计划
- 风险要给下一步处理动作

输出：
1. 本周关键结果
2. 已完成事项
3. 进行中事项
4. 风险与阻塞
5. 下周计划
```

适合：从碎片记录整理成可汇报周报。

## 12. SOP 生成

```text
请把下面这个已经跑通过的流程整理成 SOP。

输出：
1. 使用场景
2. 输入材料
3. 操作步骤
4. 每一步的检查标准
5. 常见错误
6. 输出模板
7. 下一次执行时只需要替换哪些材料
```

适合：把一次成功经验固定成可复用流程。

## 13. 风格固化

```text
请学习下面 3 个我认可的成品样例，总结它们的共同风格。

输出：
1. 语言风格
2. 结构特点
3. 常用表达
4. 禁止出现的表达
5. 后续写作规则
6. 用这个风格改写新内容时的检查清单
```

适合：让输出稳定接近个人或品牌风格。

## 14. 输出自检

```text
请检查上面的输出是否满足：

1. 是否解决了原始任务
2. 是否有编造事实
3. 是否有空泛建议
4. 是否缺少下一步动作
5. 是否能被直接复制使用
6. 是否有需要人工确认的地方

请先指出问题，再给修正版。
```

适合：所有 AI 输出进入交付前都跑一遍。

## 15. 复盘提示词

```text
请复盘这次 AI 工作流执行结果。

输入：
- 原始任务
- 使用的提示词
- AI 输出
- 人工修改
- 最终成品

请输出：
1. AI 哪些地方有效
2. 哪些地方跑偏
3. 下次提示词怎么改
4. 哪些环节应该模板化
5. 是否值得固化成工作流
```

适合：把一次运行结果变成下一版模板。

## 16. 下一步行动拆解

```text
你是一个执行计划助理。

请把下面目标拆成下一步行动。

要求：
- 不写长期愿景
- 只拆未来 48 小时能做的动作
- 每个动作必须有明确产出
- 标出最小可验证结果

输出：
1. 目标复述
2. 48 小时内最重要的 3 个动作
3. 每个动作的交付物
4. 完成标准
5. 如果卡住，下一步怎么改
```

适合：把模糊目标变成能立刻推进的任务。
